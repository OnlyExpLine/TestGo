# Notifier Service

**notifier** — это компонент для асинхронной отправки уведомлений во внешний сервис с ограничением по количеству запросов в секунду.

Решение реализует:

- worker pool
- rate limiting
- retry с экспоненциальной задержкой
- graceful shutdown
- сбор метрик

---

## Основные компоненты

**1. Notifier (notifireImpl)**

- Основная структура, реализующая интерфейс.
- Принимает сообщения через `Send`.
- Управляет worker pool.

**2. Worker Pool**

- Фиксированное количество воркеров.
- Каждый воркер читает из общего канала задач.
- Задачи обрабатываются параллельно.

**3. Queue (channel)**

- Буферизированный канал для сообщений.
- Используется для передачи задач от `Send` к воркерам.
- Обеспечивает thread-safe коммуникацию.

**4. Rate Limiter**

- Ограничивает количество запросов в секунду (N).
- Каждый worker ожидает тик перед отправкой запроса.

**5. Retry механизм**

- При получении 429 выполняется повтор.
- Используется экспоненциальный backoff.
- Максимум 3 попытки.

**6. Stats**

- Структура для сбора метрик:
  - `Sent` — успешно отправленно.
  - `Failed` — звершилось ошибкой.
  - `Retries` — количество повторных попыток.
- Обновляется атомарно для избежания гонок данных.

**7. Graceful Shutdown**

- Метод `Close()`:
  - Запрещает приём новых задач.
  - Закрывает канал очереди.
  - Дожидается завершения всех воркеров.
